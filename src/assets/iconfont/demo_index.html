Unicode Usage

Unicode is the most basic application method for fonts on the web. Its features are:

  • Supports dynamic adjustment of icon size, color, etc. as fonts.
  • By default, it does not support multi-color. Adding multi-color icons will automatically desaturate them.

Note: The new version of iconfont supports two ways to reference multi-color icons: SVG symbol reference and color font icon mode. (To use color font icons, you need to enable the "Color" option in "Edit Project" and regenerate.)

Steps to use Unicode are as follows:

Step 1: Copy the generated @font-face code from the project

@font-face {
  font-family: 'iconfont';
  src: url('iconfont.woff2?t=1713950346881') format('woff2'),
       url('iconfont.woff?t=1713950346881') format('woff'),
       url('iconfont.ttf?t=1713950346881') format('truetype');
}

Step 2: Define the styles for using iconfont

.iconfont {
  font-family: "iconfont" !important;
  font-size: 16px;
  font-style: normal;
  -webkit-font-smoothing: antialiased;
  -moz-osx-font-smoothing: grayscale;
}

Step 3: Select the corresponding icon and get its font code to apply to the page

<span class="iconfont">&#x33;</span>

"iconfont" is the font-family of your project. You can check it by editing the project; the default is "iconfont".

font-class Usage

font-class is a variant of the Unicode method, mainly to address the issues of non-intuitive writing and ambiguous semantics with Unicode.

Compared to the Unicode method, it has the following features:

  • It is more intuitive and semantically clear compared to Unicode. Easily distinguish what the icon represents.
  • Since icons are defined by classes, when you need to replace an icon, you only need to update the Unicode reference in the class.

Usage steps are as follows:

Step 1: Include the generated fontclass code from the project

<link rel="stylesheet" href="./iconfont.css">

Step 2: Select the corresponding icon and get its class name to apply it to the page

<span class="iconfont icon-xxx"></span>

"iconfont" is the font-family of your project. You can check it by editing the project; the default is "iconfont".

Symbol Usage

This is a completely new way of using icons—arguably the mainstream of the future, and the method currently recommended by the platform. For more details, refer to this article. This method essentially creates a collection of SVGs, and compared to the other two methods, it has the following features:

  • Supports multi-colored icons without being limited to a single color.
  • With certain techniques, it supports style adjustments similar to fonts using font-size and color.
  • Its compatibility is relatively poor, supporting IE9+ and modern browsers.
  • Browser rendering performance for SVG is average, not as good as PNG.

Usage steps are as follows:

Step 1: Include the generated symbol code from the project

<script src="./iconfont.js"></script>

Step 2: Add the common CSS code (include only once):

<style>
.icon {
  width: 1em;
  height: 1em;
  vertical-align: -0.15em;
  fill: currentColor;
  overflow: hidden;
}
</style>

Step 3: Select the corresponding icon and get its class name to apply it to the page

<svg class="icon" aria-hidden="true">
  <use xlink:href="#icon-xxx"></use>
</svg>
